AWS認定データエンジニアの学習を進める中で、S3のストレージクラスのユースケースがごちゃごちゃしてきたのでAIと壁打ちしながら整理しました。

## 各ストレージクラスのユースケース

### 1. S3 Standard

アクセス頻度が高いデータ向け。

- **ビッグデータ分析のホットデータ**: Amazon EMRやAthenaで毎日頻繁にクエリされるアクセスログやトランザクションデータ
- **動的ウェブサイトのコンテンツ**: 常にアクセスされるアプリの画像・動画ファイル
- **モバイルゲームのユーザーデータ**: リアルタイムで頻繁に読み書きが発生するセーブデータ

### 2. S3 Intelligent-Tiering

アクセスパターンが読めないデータ向け。自動でコスト最適なクラスに移動してくれる。

- **全社横断のデータレイク基盤**: 各部署が任意のタイミングでクエリするため、どのデータがいつアクセスされるか予測不可能な環境
- **新規サービスのログデータ**: サービス開始直後でアクセスパターンがまだ確立していない場合

### 3. S3 Standard-IA（低頻度アクセス）

アクセスは少ないが、アクセス時はミリ秒で取得できる必要があるデータ向け。

- **月次レポート用データ**: 月末のバッチ処理時だけ使うが、処理が始まったらすぐにデータが必要なログファイル
- **ディザスタリカバリのバックアップ**: 普段は使わないが、障害時には即座にリストアが必要なバックアップ

### 4. S3 One Zone-IA

Standard-IAと同様だが、データを1つのAZにしか保存しないためさらに安い。消えてもいいデータ向け。

- **再作成可能な派生データ**: 元画像からバッチ生成したサムネイル（元データから再生成できるのでAZ障害で消えてもOK）
- **オンプレのセカンダリバックアップ**: 自社サーバーに原本があり、クラウドは念のための2次バックアップ

### 5. S3 Glacier Instant Retrieval

滅多にアクセスしないが、アクセス時は即座（ミリ秒）に取得できる必要があるデータ向け。

- **医療画像や過去記事**: 古いデータだが、検索・クリックされたら瞬時に表示する必要があるもの

### 6. S3 Glacier Flexible Retrieval

数時間以内であれば取り出せれば問題ないデータ向け。

- **インシデント調査用ログ**: セキュリティインシデント時に数時間以内に取り出して分析できれば良い過去のアクセスログ
- **年次監査用データ**: 1年に1回の監査時だけ必要なバックアップ。無料のバルク取り出しでコストを抑える

### 7. S3 Glacier Deep Archive

とにかく長期保管、ほぼアクセスしないデータ向け。最安値。

- **法規制による長期保管**: 医療機関のカルテや金融機関の取引記録など、法律で7〜10年の保存が義務付けられたデータ
- **退職者のアーカイブ**: 将来的な訴訟リスクに備えて残しておく作業ファイルやメール履歴

## StandardとIntelligent-Tiering、Standard-IAの違い

ここが一番ごちゃごちゃしていたので整理しました。

選択のフロー：

- **アクセス頻度が高く、サイズもバラバラ** → S3 Standard
- **アクセス頻度は低い（月1回未満）が、ファイルサイズが大きい（128KB以上）** → S3 Standard-IA
- **アクセス頻度が全く読めず、ファイルサイズもそこそこ大きい** → S3 Intelligent-Tiering

Intelligent-TieringはStandardとStandard-IAの中間的存在で、自動でコストを最適化してくれる分、監視コストが少しかかります。ファイルサイズが小さいと監視コストが割高になるので注意。
