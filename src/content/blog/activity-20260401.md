## やったこと

### Amazon Athena と AWS Glue の違いを整理した

AWSの学習を進める中で、AthenaとGlueの役割がごっちゃになってきたので比較表を作って整理した。

| 比較項目 | Amazon Athena | AWS Glue |
|---------|--------------|----------|
| **主な役割** | データクエリ（分析・抽出） | データETL（抽出・変換・ロード）とデータカタログ管理 |
| **主な機能** | Amazon S3上のデータに対して直接標準SQLを実行する | データを変換・加工して移動させる。データの構造（スキーマ）を自動定義する |
| **裏側の技術** | Presto / Trino（分散SQLエンジン） | Apache Spark / Python（分散処理フレームワーク） |
| **得意なこと** | アドホックな分析、ログの調査、BIツール（QuickSightなど）での可視化 | データレイク・DWHの構築、データクレンジング、複雑なフォーマット変換、定期的なバッチ処理 |
| **料金体系** | クエリでスキャンしたデータ量に基づく従量課金 | ETLジョブの実行時間（リソース消費量）とカタログの保存/リクエストに基づく従量課金 |

**ざっくりまとめると：**
- **Athena** = 「S3のデータをSQLで直接読む」ツール。データを動かさずその場で問い合わせる。
- **Glue** = 「データを加工して別の場所に運ぶ」ツール。変換・整形・カタログ管理まで担う。

両者は競合ではなく、Glueでデータを整形・カタログ化して、AthenaでSQLクエリを投げる、という組み合わせで使われることが多い。
